# Anatomy of a book

Every item in your collection — whether it's a physical book, a saved PDF, or a living scout — has the same basic structure. This guide walks through what's inside a book directory, what each file does, and what you can safely change.

## The universal shape

Open any book's directory and you'll see at least these files:

```
the-dispossessed/
├── metadata.yaml       ← the catalog entry
├── README.md           ← the book's "spine" — what it is, at a glance
├── CLAUDE.md           ← instructions for Claude Code on return visits
└── context.md          ← interaction history (may not exist yet)
```

Some books have additional files depending on their type:

```
# A physical book might also have:
├── photo.jpg           ← photograph of the book

# A digital book might also have:
├── original.pdf        ← the preserved original file
├── content.md          ← readable extraction (for HTML sources)

# A scout has much more:
├── data/
│   └── entries.yaml    ← all structured data
├── docs/
│   └── getting-started.md
├── scripts/
│   ├── generate_readme.py
│   └── discover.py
```

## What each file does

### metadata.yaml — the catalog entry

This is the most important file. It contains everything the collection needs to know about the book: title, author, section, description, when it was added, what format it is, and any personal notes.

The collection index (`.collection-index.yaml` at the collection root) is built from these files. The wiki views are generated from them. When you search or browse your collection, this is the data being read.

**You can edit this file.** See the [YAML basics guide](yaml-basics.md) for how.

### README.md — the spine

If you were to pick up a book off a shelf and look at the cover and table of contents, that's what the README is. It tells you what this book is and how to navigate its contents — readable without any other context.

For physical books, the README shows the title, author, photo (if present), and metadata.
For digital books, the README shows the source, provenance, and (for HTML content) an extracted readable version.
For scouts, the README is a generated overview with categories, comparison tables, and editorial content.

**You can edit this file**, but be aware that for scouts, the README is regenerated by `scripts/generate_readme.py` and your edits will be overwritten. For other book types, your edits persist.

### CLAUDE.md — instructions for Claude Code

When you start Claude Code inside a book's directory, it reads this file automatically. It tells Claude about the book: what type it is, where the key files are, how to update it.

**You generally don't need to edit this file.** It's written by the creation skill and updated by `/coll-notes`. If Claude seems confused about a book's structure, checking CLAUDE.md is a good first step — it might be outdated.

### context.md — interaction history

This file is created the first time you use `/coll-notes` on the book. It records what you've done with the book across sessions: decisions made, user preferences observed, open questions.

It's useful when you return to a book after a long break — Claude reads it and picks up where you left off.

**You can edit this file.** Your edits persist. It's your record.

## Files you should not delete

- **metadata.yaml** — without this, the collection can't catalog the book. The wiki generator won't know it exists.
- **README.md** — without this, the wiki has nothing to display for the book.

Everything else is technically optional (the collection won't break), but:
- Deleting `CLAUDE.md` means Claude won't have operational context on return visits
- Deleting a scout's `data/entries.yaml` destroys its structured data
- Deleting a digital book's `original.pdf` removes the preserved content

## The relationship between a book and the collection

Each book is self-contained — it has everything it needs in its own directory. If you moved a book directory to a different computer, it would still be readable as a standalone set of files.

The collection adds two things on top:
1. **`.collection-index.yaml`** at the collection root — a catalog of all books, used for browsing and wiki generation. This is rebuilt from the books' individual `metadata.yaml` files, so it's never the only copy of any data.
2. **The wiki** at `wiki/` — a browseable HTML interface generated from the catalog. Also regenerable at any time.

Both are derived from the books. The books are the source of truth.
